🕐 Idle Detector
Watchdog per deployment. Tracks last-request timestamp. Triggers sleep when threshold crossed.
→
Proxy middleware stamps lastRequestAt:{id} in Redis on every request
→
BullMQ repeatable job runs every 60s · checks all RUNNING deployments
→
If now - lastRequestAt > IDLE_TIMEOUT (900s) → enqueue queue:sleep
→
Configurable per project: idleTimeoutSeconds field on Project model
Redis GETSET lastRequestAt BullMQ repeatable default 900s
🔌 Wake-Up Proxy — The Key Piece
Sits in front of SLEEPING containers. Holds incoming requests while container cold-starts. Returns a loading page or waits transparently.
1
Request arrives for {id}.domain.com
2
Proxy checks containerState:{id} in Redis → SLEEPING
3
Enqueue queue:wake job · set state → WAKING (idempotent)
4
Return HTML loading page with auto-refresh every 3s via JS polling
5
Container becomes RUNNING → proxy forwards original request
6
API clients (non-browser): 503 + Retry-After: 30 header
Redis containerState:{id} idempotent wake loading page UX 503 + Retry-After
📈 Horizontal Auto-Scaler
When a RUNNING container is under sustained load, spin up replicas. Scale back down when load drops.
→
Metric source: bare_metal = docker stats CPU%; cloud = CloudWatch ECS CPUUtilization
→
Scale-up trigger: CPU > 70% for 2 consecutive minutes → desired+1 (max 5)
→
Scale-down trigger: CPU < 20% for 5 minutes → desired-1 (min 1 while RUNNING)
→
Cloud: update ECS Service desiredCount · ALB handles routing automatically
→
Metal: docker run additional container · NGINX upstream block updated
max replicas: 5 cooldown: 120s ECS desiredCount NGINX upstream
